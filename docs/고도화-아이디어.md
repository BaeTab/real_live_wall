# real_live_wall 고도화 아이디어

> 기준 버전: **v1.1.0** (기본 씬 8종 판매급 리마스터 · 멀티모니터 · 트레이 · 자동 시작 · 설정 저장 · 헤드리스 `--screenshot` QA)
> 작성: 2026-07-03

## 북극성

경쟁자는 Steam의 **Wallpaper Engine**이다. 그들이 잘하는 것(방대한 워크샵 콘텐츠, 비디오 배경)을 정면으로 이기려 하지 말고, 우리가 이미 선택한 **차별화 3축**을 끝까지 밀어 "저 앱은 *반응한다*"는 인상을 만든다.

1. **리액티브 데이터** — 오디오·시스템뿐 아니라 *지금 듣는 곡·날씨·시간대*까지 배경이 반응.
2. **Shadertoy 무수정 호환** — 세상의 셰이더가 곧 우리 콘텐츠. 멀티패스·오디오 텍스처까지 열면 호환 범위가 폭발적으로 커진다.
3. **초저부하 & 매너** — 전체화면 게임/배터리에서 알아서 멈추는 "예의 바른" 배경. 이게 깨지면 데스크톱 앱으로서 신뢰를 잃는다.

아래는 **임팩트/노력** 기준으로 4단계로 나눴다. 각 항목에 근거·대략 난이도·아키텍처 연결점을 붙였다.

## P0 — 지금 가진 것으로 빠르게 먹는 열매 (1~3일치)

이미 있는 인프라를 조금만 확장하면 되는 것들. 특히 **`screenshot.rs` 헤드리스 렌더**가 방금 생겨서 파생 가치가 크다.

### 0-1. 씬 썸네일 자동 생성 → 인앱 갤러리
- **무엇**: `--screenshot`을 배치로 돌려 씬별 썸네일 PNG를 `assets/thumbnails/`에 굽고, egui 설정 패널의 씬 드롭다운을 **썸네일 그리드**로 교체.
- **왜**: 지금은 텍스트 목록이라 "골라 쓰는 맛"이 없다. 썸네일 갤러리는 판매 앱의 기본기이고, 이미 렌더 경로가 있어 거의 공짜.
- **연결**: `screenshot.rs`(이미 있음) → 빌드 스크립트/CI에서 전 씬 일괄 렌더 → `ui.rs`에서 `egui::ImageButton`. wgpu 텍스처로 로드.
- **난이도**: 낮음.

### 0-2. 시각 회귀 테스트(visual regression)를 CI에
- **무엇**: `--screenshot`으로 뽑은 기준 이미지와 새 렌더를 비교(픽셀 diff 임계값)해 셰이더 수정이 화면을 깨뜨리면 실패시키는 테스트.
- **왜**: 이번 리마스터에서 겪은 **sin-해시 블록 아티팩트** 같은 "컴파일은 되는데 화면이 깨지는" 회귀를 자동으로 잡는다. `cargo test`의 naga 컴파일 검증은 문법만 본다.
- **연결**: `tests/`에 헤드리스 렌더 + 이미지 해시/SSIM 비교. GPU 필요하므로 로컬/셀프호스트 러너.
- **난이도**: 낮음~중간.

### 0-3. 전체화면·배터리 자동 일시정지
- **무엇**: 포그라운드에 전체화면 창(게임/영상)이 있거나 노트북이 배터리 구동이면 렌더를 멈추거나 FPS를 확 낮춘다.
- **왜**: "초저부하"는 우리의 핵심 약속. 게임 중 GPU를 갉아먹으면 바로 삭제당한다. 데스크톱 앱 신뢰의 문제.
- **연결**: Windows `QueryFullscreenWindow` 근사(포그라운드 창 사각형 == 모니터 사각형 + `SHQueryUserNotificationState`), 배터리는 `GetSystemPowerStatus`. app.rs 프레임 루프에서 `ControlFlow::Wait` 전환 + 렌더 스킵.
- **난이도**: 낮음~중간. **강력 추천(P0 최우선)** — 제품 신뢰 직결.

### 0-4. FPS 상한 · 적응형 품질
- **무엇**: `--fps 30` 같은 상한, 그리고 프레임 시간이 예산을 넘으면 SSAA를 1.5→1.0으로 자동 강하.
- **왜**: 4K 멀티모니터에서 발열/전력 통제. 사용자가 "부드러움 vs 조용함"을 고를 수 있게.
- **연결**: `config.rs`, `app.rs` 루프 타이밍, `postfx.rs` SSAA 동적 조정(리사이즈 경로 재사용).
- **난이도**: 낮음.

### 0-5. 씬 전환 크로스페이드
- **무엇**: "다음 씬"이나 자동 순환 시 즉시 컷 대신 0.5~1s 디졸브.
- **왜**: 지금은 셰이더가 뚝 바뀐다. 부드러운 전환 하나로 체감 완성도가 크게 올라간다.
- **연결**: `renderer.rs`가 이전/다음 파이프라인 2개를 잠깐 들고 `postfx` 합성 단계에서 mix. 또는 이전 프레임을 텍스처로 잡아 페이드.
- **난이도**: 중간.

## P1 — 진짜 차별화 (핵심 투자)

여기가 "왜 이걸 써?"의 답이 나오는 구간. 리액티브 3축을 깊게 판다.

### 1-1. ⭐ 재생 곡(Now Playing) 반응 — 앨범아트 팔레트 + 곡 정보
- **무엇**: Windows **SMTC**(`GlobalSystemMediaTransportControlsSessionManager`)로 지금 재생 중인 곡의 **제목/아티스트/앨범아트**를 읽어, 앨범아트에서 **주조색 팔레트**를 추출해 셰이더 색으로 흘려보낸다. 곡이 바뀌면 배경 팔레트가 그 앨범 색으로 물든다.
- **왜**: Wallpaper Engine에도 없는 강력한 "살아있음". Spotify/유튜뮤직 아무거나 틀면 배경이 그 곡의 색으로 반응 — 데모 한 번이면 설득 끝.
- **연결**: 새 모듈 `nowplaying.rs`(WinRT `Windows.Media.Control`). 팔레트를 uniform 계약에 **신규 채널로 추가**(예: `vec4 palette[4]`) — Rust/WGSL/GLSL 3곳 동기 수정 필요(계약 확장 비용 有). 곡 정보 텍스트는 옵션 오버레이.
- **난이도**: 중상. **P1 최우선 추천.**

### 1-2. ⭐ 비트/BPM 감지 → `iBeat` / 온셋 펄스
- **무엇**: 현재는 밴드 에너지(bass/mid/treble)만 있다. **온셋 검출 + BPM 추정**을 더해 `sys.z`(이미 `beat`로 예약됨)에 비트 펄스를, 새 `iBpm`를 채운다. 셰이더가 "박자에 맞춰" 터질 수 있게.
- **왜**: 오디오 비주얼라이저의 급을 올린다. 지금은 에너지에 반응하지만 *박자*에는 못 맞춘다. 비트 싱크는 체감이 완전히 다르다.
- **연결**: `audio.rs`에 스펙트럴 플럭스 기반 온셋 + 자기상관/템포 히스토그램. uniform의 예약 필드(`sys.z`) 활용 → 계약 변경 최소.
- **난이도**: 중상(DSP).

### 1-3. Shadertoy `iChannel0` 오디오 텍스처 완전 호환
- **무엇**: Shadertoy 표준인 **512×2 오디오 텍스처**(row0=FFT, row1=파형)를 실제 텍스처로 바인딩. 그러면 "오디오 반응형 Shadertoy 셰이더"를 **거의 무수정**으로 돌린다.
- **왜**: 차별화 축 2번(호환성)의 폭을 크게 넓힌다. 세상의 오디오 셰이더가 곧 우리 콘텐츠가 된다.
- **연결**: `audio.rs`가 매 프레임 텍스처 업로드, `renderer.rs` 바인드그룹에 texture+sampler 추가, `shader.rs` 래퍼가 `iChannel0`/`texture()` 매핑. 파형 링버퍼 필요.
- **난이도**: 중상.

### 1-4. 멀티패스(버퍼) 셰이더 그래프
- **무엇**: Shadertoy `Buffer A~D` + `Image`처럼 **이전 프레임 피드백**이 가능한 다중 패스. 반응확산·유체·라이프 등 상태 기반 씬이 열린다.
- **왜**: 단일 패스로는 못 만드는 씬(누적/시뮬레이션)이 콘텐츠 상한을 결정한다. 고급 씬의 문을 여는 인프라.
- **연결**: `renderer.rs`를 패스 그래프로 일반화(핑퐁 타깃 N개, 패스별 iChannel 라우팅). 씬 정의가 단일 파일 → 매니페스트로 승격 필요(→ 2-1과 함께).
- **난이도**: 높음.

### 1-5. 마우스 인터랙션 — 커서 시차(parallax) + 클릭 파문
- **무엇**: 커서 위치로 배경이 살짝 시차 이동하고, 클릭하면 물결/파문이 퍼진다.
- **왜**: "상호작용하는 배경"은 데모에서 즐겁고 바이럴 요소. uniform에 `mouse`는 이미 있다.
- **연결(주의)**: 월페이퍼 모드에서 `WorkerW` 자식 창은 **마우스 입력을 못 받는 경우**가 많다 — 전역 `GetCursorPos` 폴링으로 우회해 uniform에 주입해야 한다. 클릭은 로우레벨 훅이 필요할 수 있어 옵트인.
- **난이도**: 중간(입력 경로가 함정).

### 1-6. 날씨 · 시간대 리액티브 씬
- **무엇**: 위치 기반 날씨(맑음/비/눈/구름) + 실제 일출·일몰 시각으로 하늘색·조도가 하루 동안 흐른다. "비 오는 날엔 rain 씬이 진짜 비처럼."
- **왜**: 창밖 대신 쓰는 배경. 앰비언트 제품의 감성 포인트.
- **연결**: 새 `ambient.rs`(비동기 HTTP, 캐시). uniform에 `weather`/`sunElev` 채널 추가. 네트워크 실패는 graceful(축 4번 원칙).
- **난이도**: 중간(비동기·API 키·프라이버시 고지).

## P2 — 제품/생태계 (판매 가능성)

콘텐츠가 늘고 남에게 권할 수 있게 만드는 층.

### 2-1. 씬 매니페스트(JSON/TOML) + 파라미터 노출
- **무엇**: 씬을 폴더+매니페스트로 정의(`scene.toml`: 이름·설명·썸네일·**튜닝 가능한 uniform 파라미터**·멀티패스 구성·필요 채널). 설정 패널이 매니페스트를 읽어 **씬별 슬라이더**를 자동 생성(색상·속도·강도).
- **왜**: 사용자가 씬을 "커스터마이즈"할 수 있어야 소장 가치가 생긴다. 멀티패스(1-4)·갤러리(2-2)의 토대.
- **연결**: `discover_scenes` 확장, 커스텀 uniform 슬롯, `ui.rs` 동적 컨트롤.
- **난이도**: 중상.

### 2-2. 씬 재생목록 · 스케줄러 · 프로필
- **무엇**: 씬 순환(N분마다/셔플), 시간대별 자동 전환(아침엔 sunset, 밤엔 aurora), 모니터별 프로필 저장.
- **왜**: "설정하고 잊는" 앰비언트 사용성. 이미 있는 트레이·config와 잘 붙는다.
- **연결**: `persist.rs` 스키마 확장, `tray.rs` 메뉴, app.rs 타이머. **모니터별 씬 선택**(로드맵 기존 항목)도 여기서 함께.
- **난이도**: 중간.

### 2-3. 인앱 온보딩 · 트레이 UX 다듬기
- **무엇**: 첫 실행 3스텝 온보딩(오디오 소스 고르기 → 씬 고르기 → "적용"), 트레이 메뉴에 씬 서브메뉴/게인 슬라이더, 토스트 알림.
- **왜**: 현재 트레이 액션 4종은 코드 검증만 됐고 실사용 온보딩이 없다. 첫인상이 리텐션을 만든다.
- **난이도**: 중간.

### 2-4. 설치 관리자 · 코드 서명 · 자동 업데이트
- **무엇**: MSI/`winget` 패키지, 서명된 exe, 인앱 업데이트 확인(GitHub Releases API).
- **왜**: 지금은 zip 수동 배포라 SmartScreen 경고가 뜬다. 서명·설치관리자·오토업데이트는 "판매 제품"의 최소 요건.
- **연결**: 릴리즈 파이프라인(이미 `gh release` 관례 있음) 확장. 서명 인증서 필요(구매/조달 이슈).
- **난이도**: 중간(+행정).

### 2-5. GIF/비디오 레이어 (선택적 차별화 축소)
- **무엇**: 절차적 셰이더 위/아래에 사용자의 GIF/MP4를 레이어로.
- **왜**: Wallpaper Engine의 주 사용처가 비디오 배경이다. 다만 우리 강점(절차·저부하)과 상충하므로 **후순위 검토**. 넣더라도 "셰이더 반응형 오버레이"로 차별화.
- **난이도**: 높음(디코더 통합·부하).

## P3 — 문샷 / 장기

### 3-1. 크로스플랫폼 표면(macOS/Linux)
- macOS: 스크린별 `kCGDesktopWindowLevel` NSWindow. Linux: X11 데스크톱 타입 / Wayland `wlr-layer-shell`. `platform.rs`는 이미 스텁·추상화가 있다. 렌더 코어(wgpu)는 그대로 재사용 가능 — **시장을 3배로.**

### 3-2. GPU 컴파일 파이프라인 상의 파티클/유체 시뮬(compute shader)
- 풀스크린 프래그먼트 한계를 넘는 수백만 파티클. wgpu compute + storage buffer. 오디오/커서 반응 파티클 필드. (성능 예산과 균형 필요.)

### 3-3. 커뮤니티 씬 공유 허브
- 매니페스트(2-1) 기반으로 GitHub 리포/간단 웹에서 씬 install. Steam Workshop의 경량 대안. 콘텐츠 플라이휠.

### 3-4. "라이브 위젯" 오버레이
- 시계·날씨·CPU·재생곡을 셰이더 위 네이티브 위젯으로. 배경+정보의 결합(앰비언트 정보 디스플레이).

### 3-5. Steam 출시
- Steam은 결제·업데이트·워크샵 인프라를 얹어준다. 2-1/3-3이 갖춰지면 진지하게 검토.

## 이번 스프린트 추천 (하이 리버리지 5개)

의존관계와 임팩트를 함께 보면 이 순서가 좋다:

1. **0-3 전체화면/배터리 자동 일시정지** — 제품 신뢰의 최소선. 가장 먼저.
2. **0-1 썸네일 자동 갤러리** — 헤드리스 렌더 재활용, 즉각적 완성도 상승.
3. **1-1 Now Playing 앨범아트 팔레트** — *가장 강한 차별화 데모*. 여기에 무게를 싣는다.
4. **1-2 비트/BPM 싱크** — 비주얼라이저의 급을 올리고 1-1과 시너지(음악 씬).
5. **2-2 재생목록·시간대 스케줄러** — "설정하고 잊는" 사용성, 기존 config/tray와 저비용 결합.

> 계약 확장 주의: 1-1·1-2·1-6은 `Uniforms` 블록(std140)에 **신규 채널**을 더한다 — Rust(`uniforms.rs`) · WGSL(`shader.rs` 기본 씬) · GLSL 래퍼(`shader.rs`) **세 곳을 바이트 단위로 함께** 고쳐야 한다. 예약 필드(`sys.z=beat`, `sys.w=fps`)를 먼저 소진하고, 남는 슬롯이 없을 때만 블록을 키우는 게 안전하다.
